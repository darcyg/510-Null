VIDEO_TOS_PRIOIRTY

Description

Get or Set ToS (Type of Service) priority configuration. The ToS setting will be set to the ToS field in IP header of every video and audio packet.

If the CHANNEL=n was not set with this URL to the Multi-Channel device, this command is read-only command and the CHANNLE 1 video server's TOS configurations will be returned. If the CHANNEL 1 video server does not connect to the router, the error message will be returned.

Input Argument

Minimize-Delay, Maximize-Throughput, Maximize-Reliability, Normal-Service

Attribute

  • Readable and Writable (except all devices in Multi-Channel Platform)
  • On-Fly (except all encoders in WPL Platform)

Return of Message

  • Read Operation
VIDEO_TOS_PRIORITY='Normal-Service'
  • Write Operation
OK: VIDEO_TOS_PRIORITY='Normal-Service'
or
ERROR: VIDEO_TOS_PRIORITY='Normal-Service'

Applicability

  • WPL Platform : All Encoder Devices
  • EPL Platform : All Encoder Devices
  • PlatformT : All encoder Devices
  • Multi-Channel Encoder : All Devices

Status

Valid

See Also

VIDEO_TOS, VIDEO_TOS_TYPE

Example

  • Read Operation
http://ip:port/cgi-bin/system?USER=Admin&PWD=123456&VIDEO_TOS_PRIORITY
or
http://ip:port/cgi-bin/cmd/system?VIDEO_TOS_PRIORITY
  • Write Operation
http://ip:port/cgi-bin/system?USER=Admin&PWD=123456&VIDEO_TOS_PRIORITY=Normal-Service
or
http://ip:port/cgi-bin/cmd/system?VIDEO_TOS_PRIORITY=Normal-Service
